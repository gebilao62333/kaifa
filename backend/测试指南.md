# 测试运行指南

## 环境要求

确保已安装以下软件：
- **Node.js**: v16.0.0 或更高版本
- **npm**: v8.0.0 或更高版本（通常随 Node.js 一起安装）

### 检查 Node.js 和 npm 是否已安装

打开终端（Windows PowerShell 或 CMD），运行以下命令：

```bash
# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version
```

如果看到版本号（例如 `v18.17.0` 和 `9.6.7`），说明已安装。

### 安装 Node.js（如果未安装）

1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 LTS（长期支持）版本
3. 运行安装程序，按照提示完成安装
4. 重新打开终端，检查版本

## 快速开始

### 步骤 1: 进入后端目录

```bash
cd c:\Users\Administrator\Desktop\kaifawenadna\backend
```

或者在 Windows 文件资源管理器中：
1. 打开 `C:\Users\Administrator\Desktop\kaifawenadna\backend`
2. 在地址栏输入 `cmd` 并按 Enter
3. 这将打开命令提示符，目录已自动切换

### 步骤 2: 安装依赖

```bash
npm install
```

这将安装 package.json 中列出的所有依赖包，包括：
- Express.js
- Socket.IO
- MySQL / MongoDB / Redis 客户端
- JWT 认证库
- 测试框架（Jest, Supertest, etc.）

**注意**: 首次安装可能需要几分钟，取决于网络速度。

### 步骤 3: 配置环境变量

```bash
# 复制示例环境配置文件
copy .env.example .env
```

编辑 `.env` 文件，配置以下内容（测试环境可以使用默认值）：

```env
# 服务器配置
PORT=3000
NODE_ENV=development

# JWT 配置
JWT_SECRET=your-secret-key-change-in-production
JWT_EXPIRES_IN=7d

# 数据库配置（测试时可使用默认配置）
DB_HOST=localhost
DB_PORT=3306
DB_NAME=duoke_peer
DB_USER=root
DB_PASSWORD=

# MongoDB
MONGO_URI=mongodb://localhost:27017/duoke_peer

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
```

### 步骤 4: 运行测试

#### 方法一: 运行所有测试

```bash
npm test
```

#### 方法二: 运行测试并生成覆盖率报告

```bash
npm run test:coverage
```

这将：
1. 运行所有测试用例
2. 生成详细的覆盖率报告
3. 在终端显示覆盖率统计
4. 创建 HTML 报告（可在浏览器中查看）

#### 方法三: 监听模式（开发时使用）

```bash
npm run test:watch
```

文件更改时自动重新运行测试。

## 测试命令详解

### npm test
```bash
npm test
```
运行所有测试文件（*.test.js 和 *.spec.js）

### npm run test:coverage
```bash
npm run test:coverage
```
运行测试并生成覆盖率报告，包含：
- **文本报告**: 终端中显示
- **HTML 报告**: `coverage/lcov-report/index.html`
- **JSON 摘要**: `coverage/coverage-summary.json`

### npm run test:watch
```bash
npm run test:watch
```
监听文件变化，自动重新运行测试（适合开发阶段）

### npm run test:verbose
```bash
npm run test:verbose
```
详细输出模式，显示每个测试的名称和状态

## 常见问题解决

### 问题 1: npm install 失败

**症状**: 安装依赖时出现错误

**解决方案**:
```bash
# 清除 npm 缓存
npm cache clean --force

# 删除 node_modules 重新安装
rmdir /s /q node_modules
npm install
```

### 问题 2: 测试找不到模块

**症状**: `Cannot find module`

**解决方案**:
```bash
# 确保在正确目录
cd c:\Users\Administrator\Desktop\kaifawenadna\backend

# 重新安装依赖
npm install
```

### 问题 3: Jest 配置错误

**症状**: Jest 相关错误

**解决方案**:
```bash
# 检查 jest.config.js 存在
# 确保 jest 已安装
npm install jest --save-dev
```

### 问题 4: 测试超时

**症状**: 测试运行时间过长或超时

**解决方案**:
编辑 `jest.config.js`:
```javascript
module.exports = {
  testTimeout: 30000, // 增加超时时间
  // ...
};
```
