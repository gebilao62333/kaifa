# 多客陪玩平台 - 数据库设计文档

## 一、数据库概述

### 1.1 数据库架构

| 数据库 | 用途 | 存储引擎 |
|--------|------|----------|
| MySQL | 核心业务数据 | InnoDB |
| MongoDB | 聊天消息、日志 | - |

### 1.2 命名规范

- 表名统一使用 `xn_` 前缀
- 字段名使用下划线命名法 (snake_case)
- 索引名格式: `idx_表名_字段名`
- 主键统一使用 `id`，自增 BIGINT 类型

## 五、数据库优化建议

### 5.1 索引优化

1. **高频查询字段添加索引**: user_id, create_time, status
2. **唯一约束字段添加唯一索引**: mobile, open_id, order_no
3. **联合索引优化**: (user_id, status), (target_user_id, create_time)

### 5.2 分库分表建议

1. **聊天消息表**: 建议按月分表，避免数据量过大
2. **聊天记录表**: 建议定期归档或迁移到MongoDB
3. **游戏订单表**: 历史数据归档到冷库

### 5.3 缓存策略

1. **用户信息**: Redis缓存，TTL 1小时
2. **游戏分类**: Redis缓存，TTL 24小时
3. **礼物列表**: Redis缓存，TTL 12小时
4. **VIP套餐**: Redis缓存，TTL 24小时

### 5.4 数据安全

1. **敏感字段加密**: password字段使用bcrypt加密
2. **定期备份**: 每日全量备份，每小时增量备份
3. **权限控制**: 遵循最小权限原则
