## 1. 概述

本文档定义了多客陪玩/社交娱乐平台的统一数据字段命名规范，确保前后端数据结构一致性。



## 2. 字段命名规范

### 2.1 命名规则
- **小驼峰命名法**：如 `userId`, `userName`, `createTime`
- **避免缩写**：使用完整单词，如 `balance` 而非 `bal`
- **语义清晰**：字段名应准确表达含义

### 2.2 通用字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int/bigint | 主键ID |
| createTime | int/datetime | 创建时间 |
| updateTime | int/datetime | 更新时间 |
| status | tinyint | 状态(0/1) |
