## 🎯 三、核心特性

### 1. API服务架构

**分层设计:**
```
视图层 (Vue Components)
    ↓
服务层 (Services) - API调用
    ↓
工具层 (Common) - HTTP请求封装
    ↓
后端API
```

**特点:**
- ✅ 统一的错误处理
- ✅ 参数验证
- ✅ Token自动管理
- ✅ 401自动跳转登录
- ✅ 请求超时处理
- ✅ Loading状态管理

### 2. 状态管理 (Pinia)

**Store设计:**
```javascript
state: {
  token: '',
  profile: null,
  loading: false
}

getters: {
  isLogin,
  userId,
  nickName,
  ...
}

actions: {
  login(),
  logout(),
  fetchUserInfo(),
  ...
}
```

### 3. 登录流程

```
用户输入账号密码
    ↓
表单验证
    ↓
调用 userStore.login()
    ↓
发送API请求
    ↓
保存Token到localStorage
    ↓
保存用户信息到Store
    ↓
显示成功提示
    ↓
跳转到目标页面
```

### 4. 首页数据流程

```
页面加载 (onMounted)
    ↓
并发请求 (Promise.all)
  - loadBanners() - Banner数据
  - loadRecommendCompanions() - 推荐列表
    ↓
处理响应
  - 成功: 更新UI
  - 失败: 显示Mock数据
    ↓
渲染页面
```

### 1. API调用规范

```javascript
// ✅ 标准API调用
try {
```

## 📝 八、注意事项

### 1. Mock数据降级

所有API调用都应包含Mock数据降级:
```javascript
try {
  const result = await api.getData()
  // 处理真实数据
} catch (error) {
  // 降级到Mock数据
  this.data = mockData
}
```

### 2. 错误处理

始终使用try-catch:
```javascript
try {
  await userStore.login(credentials)
} catch (error) {
  console.error('登录失败:', error)
  toast.error('登录失败，请重试')
}
```

### 3. 加载状态

使用loading状态提升用户体验:
```javascript
isLoading.value = true
try {
  // 业务逻辑
} finally {
  isLoading.value = false
}
```
