## 🎯 三、技术架构

### 3.1 分层架构

```
表现层 (Views)
    ↓
业务逻辑 (Stores)
    ↓
服务层 (Services) ← 今日重点
    ↓
工具层 (Common/Utils)
    ↓
后端API (已启动: http://localhost:3000)
```

### 3.2 核心技术栈

- ✅ **Vue 3 Composition API** - 现代化的组件开发
- ✅ **Pinia** - 轻量级状态管理
- ✅ **Socket.IO** - 实时双向通信
- ✅ **Axios** - HTTP请求封装
- ✅ **Async/Await** - 异步编程范式
- ✅ **TypeScript类型检查** - 类型安全

### 3.3 设计模式

- **分层设计**: 清晰的分层架构
- **单一职责**: 每个文件专注一个功能
- **依赖注入**: 通过import解耦
- **响应式编程**: Vue 3 reactivity
- **事件驱动**: Socket.IO事件通信

validateParams(params, {

### 5.2 错误处理

✅ **全局错误处理**
```javascript
try {
  const result = await api.method()
  if (result.code === 200) {
    // 成功
  } else {
    toast.error(result.message)
  }
} catch (error) {
  console.error('请求失败:', error)
  toast.error('网络错误')
}
```

✅ **Mock数据降级**
```javascript
try {
  const result = await api.getData()

} catch (error) {
  data.value = mockData
}
```

- 当前使用 **Mock模式** (开发模式)
